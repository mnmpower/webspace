Stage - Applicatie met nummerplaatherkenning

Bij Rhea N.V.

2 Maart 2020 - 29 mei 2020

Tijdens het laatste jaar van mijn studies in de ITFactory van Thomas More liep ik gedurende 13 weken stage bij het bedrijf Rhea N.V. Dat is een klein bedrijf met een 15-tal werknemers. Het is lid van de Cronos Groep en bestaat al meer dan 20 jaar. Voorlopig houdt het zich bezig met drie grote pijlers:

  • Applicatie-ontwikkeling met IOT, AI en Robot gestuurde procesautomatisering
  • Herschrijven of herontwikkelen van oude applicaties
  • Onderhoud van zelfgemaakte applicaties

De opdracht

Binnen het bedrijf Rhea NV was er de nood om een applicatie te maken om problemen die zich voordoen op de parking vlotter af te kunnen handelen. Het bleef maar bij een idee omdat er nog nooit iemand aan begonnen was. Mijn stage bestond er dus uit om deze 'proof of concept' te bestuderen en daarna af te maken.

Analysefase

Ik begon dus met verschillende delen van het project te analyseren. Eerst ben ik op zoek gegaan naar een goed softwarepakket dat me kon helpen bij de nummerplaatherkenning. Dit heb ik dan onmiddelijke in de praktijk beginnen testen. Zo heb ik stuk voor stuk toegewerkt naar mijn plaatherkennigns API.

Vervolgens ben ik me gaan verdiepen in het Vidyano framework. Omdat dit framework redelijk centraal staat in Rhea NV was dit de basis van mijn project en moest ik zorgen dat alle andere componenten hiermee konden samen werken. Vidyano zelf ondersteunt al enorm veel verschillende componenten die ik kon gebruiken waardoor het bijna niet meer nodig was om nog andere dingen te integreren.

Realisatiefase

De uitwerking van mijn opdracht omvat op dit punt een Docker container met daarin de API die ik gemaakt heb. In deze Docker container heb ik ook de software geïnstaleerd voor mijn nummerplaatherkenning. Dit is een programa dat bij opstarten één foto binnen krijgt en daarvan een output met mogelijkse nummerplaten teruggeeft. Telkens als je de API aanspreekt, start deze dit programma en vangt de respons van de software op. Op zijn beurt geeft de API deze als zijn antwoord terug door.

Daarnaast heb ik ook nog de applicatie in het Vidyano framework gemaakt. Deze bevat de database met alle gegevens die de applicatie nodig heeft. Vidyano maakt gebruik van webcomponents op basis van 'Polymer'. Deze heb ik extra opmaak gegeven met de Bootstrap library. De applicatie zelf spreekt de achtercamera van het toestel aan. Deze begint te filmen en stuurt op regelmatige basis screenshots van de video door naar de API. Zodra deze een geldige respons teruggeeft stopt de camera met filmen. Dan kan de gebruiker nog een melding selecteren of invullen en wordt er een mail gestuurd naar de eigenaar van de bedrijfswagen met deze melding. Zo kan die notie nemen van het voorval.

Stageverloop

Omdat ik een goede start heb genomen en een diepgaande analyse gedaan heb kon ik vlot starten met de realisatiefase van mijn stage. Daardoor was de uitwerking van mijn stageopdracht snel klaar. We hebben de opdracht dan uitgebreid met enkele handige extra’s. Eén daarvan was de fotoknop te vervangen door de filmende camera die constant foto’s doorstuurt tot een geldige nummerplaat gevonden wordt. Zo moet de gebruiker niet constant de camera heropstarten om een foto te trekken als er geen nummerplaat herkend wordt.

Nadat ik nog een aantal andere kleine verbeteringen heb aangebracht aan de applicatie moesten we wachten op de update van het framework. Deze update hield in dat Vidyano volledig compatibel is met Docker. Na deze update moest ik dus heel de applicatie overzetten en laten werken in Docker containers.

Tijdens de periode dat ik moest wachten op de update heb ik mezelf verdiept in het gebruiken van Docker containers en meer bepaald hoe je verschillende containers kan laten samenwerken via Docker-compose. Dit heb ik gedaan door een testproject volledig Docker-compatibel te maken en uit te zoeken hoe ik dit kan hosten.

Na de update heb ik heel mijn applicatie overgezet en volledig laten werken in Docker. Dit was dan ook het einde van mijn stage.

Stage - Applicatie met nummerplaatherkenning

2 Maart 2020 - 29 mei 2020

Lees meer over mijn stage.

Tijdens het laatste jaar van mijn studies in de ITFactory van Thomas More liep ik gedurende 13 weken stage bij het bedrijf Rhea N.V. Dat is een klein bedrijf met een 15-tal werknemers. Het is lid van de Cronos Groep en bestaat al meer dan 20 jaar.

Binnen het bedrijf Rhea NV was er de nood om een applicatie te maken om problemen die zich voordoen op de parking vlotter af te kunnen handelen. Het bleef maar bij een idee omdat er nog nooit iemand aan begonnen was. Mijn stage bestond er dus uit om deze 'proof of concept' te bestuderen en daarna af te maken.
Ik ben begonnen met verschillende delen van het project te analyseren. Eerst ben ik op zoek gegaan naar een goed softwarepakket dat me kon helpen bij de nummerplaatherkenning. Dit heb ik dan onmiddelijk in de praktijk beginnen testen. Vervolgens ben ik me gaan verdiepen in het Vidyano framework. Omdat dit framework redelijk centraal staat in Rhea NV was dit de basis van mijn project. Alle andere componenten moesten hier dus mee kunnen samen werken.
De applicatie bestaat uit drie Docker containers. Deze kan je opstarten door middel van één commando. De eerste container bevat een mssql server en database. In de tweede container draait het front-end programma van Vidyano. De layout hiervan is gemaakt met Polymer webcomponenten met behulp van de Bootstrap library. De derde container bevat de alpr software en de zelfgeschreven .NET CORE api om deze aan te spreken.
Omdat ik een goede start heb genomen en een diepgaande analyse gedaan heb kon ik vlot starten met de realisatiefase van mijn stage. Daardoor was de uitwerking van mijn stageopdracht vroegtijdig klaar. We hebben de opdracht dan uitgebreid met enkele handige extra’s.
  • De knop om een foto te trekken is vervangen door een filmende camera die op regelmatige basis zelf een foto doorstuurt tot er een geldige nummerplaat gevonden is.
  • Na de update van het Vidyano framework heb ik de applicatie volledig cross platform gemaakt en in docker containers gestoken om deze eenvoudig op te kunnen starten.
Het laatste deel van mijn stage was het maken van de gebruikers- en installatiehandleiding.

Project 4.0 - Monitoring van de veiligheid rond een bewegende last

20 Januari 2020 - 28 Februari 2020

Lees meer over mijn project.

Tijdens de lessen van "Project 4.0” in 3 ITF heb ik als deel van een multidisciplinair team gewerkt aan een project om de veiligheid in het staalbedrijf Aperam te verbeteren. Dit project moest een totaaloplossing worden die zowel de veiligheid verbeterde als de nodige overtredingen bijhield.

Binnen het bedrijf Aperam is er een bewegende brugkraan. Dit is een enorme kraan die over het grootste deel van de werkvloer kan bewegen. Deze kraan kan een zware ballast dragen en is nodig voor de werking van het bedrijf. Het is uiteraard niet veilig om hier onderdoor te lopen maar dit gebeurt toch. Omdat dit niet mag volgens de veiligheidsvoorschriften is er gevraagd of we een oplossing konden bedenken om de werkvloer te monitoren en deze dan ook te realiseren.
Voor dit project besteedde mijn team acht weken aan de analysefase. Elke week kregen we één namiddag om hieraan te werken. Iedereen had dan opzoekwerk gedaan over één bepaald aspect. We zaten dan samen om dit uitvoerig te bespreken. Dit schreef ik dan neer in een analysedocument. Na de analysefase kregen we vier weken tijd om dit project effectief uit te werken. Ik heb dus ook twee aparte GitHub repositories aangemaakt om de projecten op een goede manier bij te houden.
Mijn bijdrage aan dit project was het maken van de applicatie. Als frontend framework heb ik gekozen voor Angular. De bijhorende API heb ik geschreven in asp.NET. De applicatie zelf bestaat uit twee delen. Het eerste deel is een dashboard met alle overtredingen van het personeel. Het tweede deel is de beheerderskant waarop alles nauwgezet opgevolgd kan worden. Zo heeft de beheerder een live overzicht van de gevarenzonen waar hij de werknemers kan volgen en een pagina waar alle overtreding op staan.
